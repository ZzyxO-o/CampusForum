# CampusForum 代码风格指南

## 概述
本文档定义了CampusForum项目的代码风格规范，所有开发人员必须遵循这些规范。

## Controller层风格

### 基本规范
- 使用`@Slf4j`注解记录日志
- 使用`@RestController`和`@RequestMapping`定义控制器
- 使用`@Tag`注解定义接口分组
- 使用`@Operation`注解描述接口功能
- 使用`@Parameter`注解描述路径参数

### 方法命名规范
- 查询：`get` + `资源名`（如：getDiscussions）
- 创建：`create` + `资源名`（如：createDiscussion）
- 更新：`update` + `资源名`（如：updateDiscussion）
- 删除：`delete` + `资源名`（如：deleteDiscussion）

### 日志记录规范
- 在方法开始处使用`log.info`记录方法调用
- 日志格式：`log.info("操作描述: {}", 参数对象)`
- 示例：`log.info("发布讨论: {}", discussionDto)`

### 返回值规范
- 成功响应：`Result.success(数据)`
- 失败响应：由全局异常处理器处理
- 创建操作返回ID：`Result<DiscussionMessageVo>`
- 查询操作返回分页：`Result<PageResult>`

### 示例代码
```java
@Slf4j
@RestController
@RequestMapping("/api/discussions")
@Tag(name = "讨论管理", description = "讨论帖相关操作")
public class DiscussionController {

    @Resource
    private DiscussionService discussionService;

    @GetMapping
    @Operation(summary = "获取最新讨论列表", description = "分页获取最新讨论帖列表，支持搜索和筛选")
    public Result<PageResult> getDiscussions(DiscussionPageQueryDto queryDto) {
        log.info("获取最新讨论列表: {}", queryDto);
        return Result.success(discussionService.pageQueryDiscussions(queryDto));
    }

    @PostMapping
    @Operation(summary = "发布讨论", description = "创建新的讨论帖")
    public Result<DiscussionMessageVo> createDiscussion(@RequestBody DiscussionDto discussionDto) {
        log.info("发布讨论: {}", discussionDto);
        return Result.success(discussionService.createDiscussion(discussionDto));
    }
}
```

## Service层风格

### 接口定义规范
- 使用清晰的JavaDoc注释
- 包含`@param`和`@return`说明
- 方法命名与Controller层保持一致

### 示例代码
```java
public interface DiscussionService extends IService<Discussion> {

    /**
     * 分页查询讨论列表
     * @param queryDto 查询条件
     * @return 分页结果
     */
    PageResult pageQueryDiscussions(DiscussionPageQueryDto queryDto);

    /**
     * 创建新讨论
     * @param discussionDto 讨论信息
     * @return 创建结果（包含讨论ID和消息）
     */
    DiscussionMessageVo createDiscussion(DiscussionDto discussionDto);

    /**
     * 更新讨论
     * @param discussionUpdateDto 更新信息
     */
    void updateDiscussion(DiscussionUpdateDto discussionUpdateDto);

    /**
     * 删除讨论
     * @param discussionId 讨论ID
     */
    void deleteDiscussion(Long discussionId);
}
```

## ServiceImpl层风格

### 基本规范
- 使用`@Service`注解
- 使用`@Override`注解重写方法
- 使用`@Resource`注入依赖
- 使用`BeanUtils.copyProperties`复制属性
- 使用异常类抛出业务异常

### 异常处理规范
- 检查资源是否存在：`if (resource == null)`
- 抛出业务异常：`throw new BusinessException("错误信息")`
- 不使用try-catch捕获异常（由全局异常处理器处理）

### 时间处理规范
- 使用`LocalDateTime.now()`获取当前时间
- 设置创建时间：`entity.setCreatedTime(LocalDateTime.now())`
- 设置更新时间：`entity.setUpdatedTime(LocalDateTime.now())`

### 用户ID获取规范
- 使用`ThreadLocalUtil.getCurrentId()`获取当前登录用户ID
- 示例：`discussion.setUserId(ThreadLocalUtil.getCurrentId())`

### 示例代码
```java
@Service
public class DiscussionServiceImpl extends ServiceImpl<DiscussionMapper, Discussion> implements DiscussionService {

    @Resource
    private DiscussionMapper discussionMapper;

    @Override
    public DiscussionMessageVo createDiscussion(DiscussionDto discussionDto) {
        // 创建实体对象
        Discussion discussion = new Discussion();
        
        // 复制属性
        BeanUtils.copyProperties(discussionDto, discussion);
        
        // 设置用户ID
        discussion.setUserId(ThreadLocalUtil.getCurrentId());
        
        // 设置默认值
        discussion.setStatus("active");
        discussion.setViewCount(0);
        discussion.setReplyCount(0);
        discussion.setLikeCount(0);
        discussion.setFavoriteCount(0);
        
        // 设置时间
        discussion.setCreatedTime(LocalDateTime.now());
        discussion.setUpdatedTime(LocalDateTime.now());
        
        // 插入数据库
        discussionMapper.insert(discussion);
        
        // 返回结果
        DiscussionMessageVo discussionMessageVo = new DiscussionMessageVo();
        discussionMessageVo.setDiscussionId(discussion.getId());
        discussionMessageVo.setMessage("讨论创建成功");
        return discussionMessageVo;
    }

    @Override
    public void deleteDiscussion(Long discussionId) {
        // 检查资源是否存在
        Discussion discussion = discussionMapper.selectById(discussionId);
        if (discussion == null) {
            throw new DiscussionQueryException("讨论不存在");
        }
        
        // 软删除：修改状态
        discussion.setStatus("deleted");
        discussion.setUpdatedTime(LocalDateTime.now());
        discussionMapper.updateById(discussion);
    }
}
```

## 查询条件构建规范

### MyBatis-Plus QueryWrapper使用
- 使用`QueryWrapper`构建查询条件
- 使用链式调用构建条件
- 注意字段名使用下划线格式

### 示例代码
```java
QueryWrapper<Discussion> wrapper = new QueryWrapper<>();

// 等值判断
if (queryDto.getCategory() != null && !"all".equals(queryDto.getCategory())) {
    wrapper.eq("category", queryDto.getCategory());
}

// 模糊查询
if (queryDto.getKeyword() != null && !queryDto.getKeyword().isEmpty()) {
    wrapper.and(w -> w.like("title", queryDto.getKeyword())
                    .or()
                    .like("content", queryDto.getKeyword()));
}

// 排序
wrapper.orderByDesc("created_time");

// 状态过滤
wrapper.eq("status", "active");
```

## VO对象规范

### 返回值对象设计
- 创建操作：返回包含ID和消息的VO
- 查询操作：返回分页结果或详情VO
- 使用清晰的字段命名

## DTO对象规范

### 请求参数对象设计
- 使用清晰的字段命名
- 提供合理的默认值
- 使用`@Data`注解简化代码

### 示例代码
```java
@Data
public class DiscussionPageQueryDto {
    private Integer page = 1;        // 默认第一页
    private Integer size = 10;       // 默认每页10条
    private String category = "all";   // 默认全部
    private String tags;              // 标签筛选
    private String keyword;           // 关键词搜索
}

@Data
public class DiscussionDto {
    private String title;      // 讨论标题
    private String content;     // 讨论内容
    private String category;    // 分类
    private String tags;        // 标签
}
```

## 通用规范

### 导入顺序
1. Java标准库
2. 第三方库（如：lombok, jakarta, springframework）
3. 项目内部包（如：cn.zuo.dto, cn.zuo.service）

### 注解使用
- Controller：`@Slf4j`, `@RestController`, `@RequestMapping`, `@Tag`, `@Operation`, `@Parameter`
- Service：`@Service`, `@Resource`, `@Override`
- Mapper：`@Mapper`, `@Resource`

### 变量命名
- 驼峰命名法：`discussionService`, `discussionDto`, `discussionPageQueryDto`
- 布尔值：`isLiked`, `isActive`
- 列表：`discussions`, `replies`, `users`

### 常量定义
- 使用大写字母和下划线：`ACTIVE_STATUS`, `DEFAULT_PAGE_SIZE`
- 避免魔法值：使用常量替代硬编码

## 注意事项

1. **不要使用try-catch捕获异常**
   - 让异常抛出到全局异常处理器
   - 保持代码简洁

2. **使用BeanUtils.copyProperties复制属性**
   - 避免手动设置每个字段
   - 提高代码可维护性

3. **使用ThreadLocalUtil.getCurrentId()获取用户ID**
   - 不要从请求参数中获取
   - 确保安全性

4. **使用LocalDateTime.now()设置时间**
   - 不要使用new Date()
   - 使用Java 8的日期时间API

5. **保持方法命名一致性**
   - Controller、Service、ServiceImpl保持一致
   - 使用统一的命名规范

6. **添加清晰的注释**
   - 接口方法必须有JavaDoc注释
   - 复杂逻辑需要行内注释
